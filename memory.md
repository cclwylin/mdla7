# MDLA7 Memory: High-Level Module Spec

This file is the short memory version. For detailed ports/descriptors, see
`spec.md`.

## Global Rules

- Final target is FPGA. veriog 版本要可合成到 FPAG
- Active RTL tree is `rtl/verilog`; do not recreate `verilog_ctrl` /
  `verilog_final`.
- Final Verilog dataflow must stay:
  `DRAM -> UDMA -> L1 -> Engine/Postprocess -> L1 -> UDMA -> DRAM -> check`.
  For CONV, postprocess means `CONV psum/acc -> Requant -> L1`.
- Engines must not secretly read files or bypass Host / Command / UDMA /
  L1Manager / L1Mesh control.
- `--option dpi` may accelerate arithmetic only; it must not bypass the real
  control/data path.
- Verilog mode should be more RTL than CX mode. CX may model abstract route/resource
  timing; Verilog targets should implement explicit hardware structure such as
  arbiters, FIFOs, routers, links, and handshakes.
- Current engine coverage is mostly sample/tilelet sized. Do not tune
  synth/verilog cycle ratio until compact tile/full traversal exists.
- Regression PASS/ok means full original graph coverage plus compiled graph
  verification. Unsupported/skipped original TFLite ops are failures, not
  warnings; `compile-skipped:N` must not be counted as a clean pass.
- `matrlz`/materialized fallback layers are correctness coverage boundaries,
  not proof of native MDLA7 datapath or performance coverage. Use the
  unsupported-op audit's `--strict-native` mode when native-only coverage is
  required.
- BMM and ETHZ Verilog runs must include materialized-boundary SRAM CRC checks;
  a stale `.bin` generated by an older compiler must never be accepted as PASS
  after current `compile_model.py` fails.
- BMM Verilog closed-loop must require full final output coverage. Sampled or
  partial final coverage is a failure for BMM.
- SystemC layer PASS is a per compiled-layer DRAM reference compare. It is not
  by itself proof that the original TFLite final output was checked when any
  original op was skipped or fused without a materialized compare boundary.
- Compiler binary ops must consume real compatible constant tensor inputs when
  present. Falling back to deterministic synthetic input is only acceptable
  when the constant cannot be reshaped/broadcast to the compiled fallback
  shape.
- Large FP binary scalar-B payloads may use the compact scalar-broadcast marker
  in the SystemC program image. This is a storage/correctness optimization,
  not a Verilog-native performance claim.
- Do not skip a producer DRAM store when the next large FP binary/D2SPACE
  consumer reloads its aliased input from DRAM. A 1-byte stream barrier in that
  buffer corrupts the consumer by exactly one byte while looking almost
  correct.
- Explicit `OK_MATERIALIZE` fallback is a correctness boundary, not a
  transparent view. Stage its reference bytes at `dram_out` and emit a self-copy
  descriptor; staging at `dram_in` can alias the previous layer output and be
  overwritten before the materialized layer runs.
- `OK_MATERIALIZE` producers must not be marked producer-no-store. Binary EWE
  consumers (`ADD`, `MUL`, `SUB`) also need the preceding DRAM boundary unless a
  proven operand-forwarding path exists; BMM attention scale+mask failed when a
  `MUL` store was suppressed before `ADD`.
- Current forced materialized native exceptions include
  `inception_v3_float` L2/L5, `unet_quant` L8/L14, `unet_float` L1-L13,
  `imdn_float` L1, `dped_float` L1, and `efficientnet_b4_float` L473/L474.
- Verilog host `measured_cycle_count` must include probe descriptors. BMM full
  coverage uses final/check probe descriptors; excluding probes made
  `verilog_cyc=0` and a fake `0.00x` ratio even though the closed-loop path ran.

## Module Bus

| Block / Link                  | Drawio bus groups                                         |
| ----------------------------- | --------------------------------------------------------- |
| Host -> Command Engine        | descriptor `1T`                                         |
| Command Engine -> each engine | config `W (1T)`                                         |
| CONV ACT read                 | `32 x 16B R`, direct L1Mesh -> CONV                     |
| CONV WGT read                 | `32 x 16B R`, direct L1Mesh -> CONV                     |
| CONV writeback                | no target Payload W to L1; psum/acc must chain to Requant |
| CONV -> Requant chain         | dedicated `4096 bit/cycle` INT32/acc chain              |
| Requant                       | `8 x 16B R` + `8 x 16B W`                             |
| EWE                           | `16 x 16B R` + `8 x 16B W`                            |
| POOL                          | `16 x 16B R` + `8 x 16B W`                            |
| TNPS                          | `8 x 16B R` + `8 x 16B W`                             |
| L1Manager <-> L1Mesh          | `16 x 16B R` + `16 x 16B W`                           |
| UDMA <-> L1Manager            | `16 x 16B R` + `16 x 16B W`                           |
| UDMA <-> DRAM                 | 4 x 16B AXI R/W                                           |
