# MDLA7 Memory: 高階模組記憶

這份檔案是短版專案記憶。詳細 port、descriptor、硬體規格請看
`spec.md`。

## 全域規則

- 最終目標是 FPGA，Verilog 版本要能合成到 FPGA。
- 目前有效的 RTL tree 是 `rtl/verilog`；不要再重建 `verilog_ctrl` /
  `verilog_final` 這種舊切分。
- 最終 Verilog dataflow 必須維持：
  `DRAM -> UDMA -> L1 -> Engine/Postprocess -> L1 -> UDMA -> DRAM -> check`.
  對 CONV 來說，postprocess 指的是 `CONV psum/acc -> Requant -> L1`。
- **CONV/DWCONV/FC 永遠不直接寫 L1**。任何 dtype（INT8/INT16/FP）的 conv-class
  輸出都走 `4096 bit/cyc` chain 到 Requant，由 Requant 負責 L1 寫回：
  INT path 做 bias + MBQM + zp + clamp，FP path 做 bias + clamp + FP16 cast，
  兩者共用同一條 `8 x 16B W` Payload W lane。Sample/bring-up 例外只能用於除錯，
  不可進 production datapath。
- Engine 不可以偷偷讀檔，也不可以繞過 Host / Command / UDMA /
  L1Manager / L1Mesh control path。
- `--option dpi` 只能加速 arithmetic；不能繞過真實 control/data path。
- Verilog mode 要比 CX mode 更接近 RTL。CX 可以抽象化 route/resource timing；
  Verilog 要逐步落到 arbiter、FIFO、router、link、handshake 這些明確硬體結構。
- 目前 engine coverage 多數仍是 sample/tilelet 尺度。compact tile 或 full
  traversal 還沒完整前，不要急著調 synth/verilog cycle ratio。
- Regression 的 PASS/ok 代表「原始 graph 全覆蓋」加上「compiled graph 有驗證」。
  unsupported/skipped 原始 TFLite op 是 failure，不是 warning；
  `compile-skipped:N` 不能算 clean pass。

## 模組匯流排

| Block / Link                  | Drawio bus groups                                         |
| ----------------------------- | --------------------------------------------------------- |
| Host -> Command Engine        | descriptor `1T`                                         |
| Command Engine -> each engine | config `W (1T)`                                         |
| CONV ACT read                 | `32 x 16B R`, direct L1Mesh -> CONV                     |
| CONV WGT read                 | `32 x 16B R`, direct L1Mesh -> CONV                     |
| CONV writeback                | **no Payload W to L1 for any dtype** (INT8/INT16/FP all chain to Requant) |
| CONV -> Requant chain         | dedicated `4096 bit/cycle` INT32/acc chain (FP path bit-casts on same lanes) |
| Requant (INT mode)            | bias + MBQM(scale/shift) + zp + clamp -> INT8/INT16, writes via `8 x 16B W` |
| Requant (FP mode)             | bias + clamp + FP32->FP16 cast (no MBQM), writes via the same `8 x 16B W` |
| Requant                       | `8 x 16B R` + `8 x 16B W` (params/readback R, output W shared by both modes) |
| EWE                           | `16 x 16B R` + `8 x 16B W`                            |
| POOL                          | `16 x 16B R` + `8 x 16B W`                            |
| TNPS                          | `8 x 16B R` + `8 x 16B W`                             |
| L1Manager <-> L1Mesh          | `16 x 16B R` + `16 x 16B W`                           |
| UDMA <-> L1Manager            | `16 x 16B R` + `16 x 16B W`                           |
| UDMA <-> DRAM                 | 4 x 16B AXI R/W                                           |
